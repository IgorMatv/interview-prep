unit

Unit tests + Mockito

@ExtendWith(MockitoExtension.class) class RouteServiceTest { @Mock RouteRepository repo; @Mock RoutingStrategy strategy; @InjectMocks RouteService service; @Test void optimize_returnsOptimizedRoute() { // given var pois = List.of(new Poi(1L), new Poi(2L)); Mockito.when(strategy.optimize(pois)).thenReturn(pois); // when var result = service.optimize(pois); // then Assertions.assertEquals(2, result.size()); Mockito.verify(strategy).optimize(pois); } @Test void optimize_emptyList_throwsException() { Assertions.assertThrows( IllegalArgumentException.class, () -> service.optimize(List.of()) ); } }
@InjectMocks — Mockito сам вприскує моки. Але якщо сервіс використовує constructor injection (рекомендовано) — @InjectMocks може не спрацювати з деякими версіями. Явний конструктор надійніший: new RouteService(repo, strategy).

@WebMvcTest — тест контролера

Завантажує тільки web layer: контролери, фільтри, конвертори. Сервіси — мокаються. Швидше ніж @SpringBootTest.

@WebMvcTest(RouteController.class) class RouteControllerTest { @Autowired MockMvc mockMvc; @MockBean // mock Spring bean, не Mockito RouteService service; @Autowired ObjectMapper mapper; @Test void createRoute_validRequest_returns201() throws Exception { var request = new RouteRequest(List.of(1L, 2L)); var response = new RouteDto(42L); Mockito.when(service.create(request)).thenReturn(response); mockMvc.perform(MockMvcRequestBuilders .post("/api/routes") .contentType(MediaType.APPLICATION_JSON) .content(mapper.writeValueAsString(request))) .andExpect(MockMvcResultMatchers.status().isCreated()) .andExpect(MockMvcResultMatchers.jsonPath("$.id").value(42)); } }
E2E

@SpringBootTest — повний контекст

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT) @ActiveProfiles("test") class RouteApiIntegrationTest { @Autowired TestRestTemplate restTemplate; @Autowired RouteRepository repo; @Test void createAndFetchRoute() { var request = new RouteRequest(List.of(1L, 2L)); var created = restTemplate.postForEntity( "/api/routes", request, RouteDto.class); Assertions.assertEquals(HttpStatus.CREATED, created.getStatusCode()); Assertions.assertTrue(repo.existsById(created.getBody().getId())); } } // WebEnvironment варіанти: // RANDOM_PORT: реальний embedded server, рандомний порт // DEFINED_PORT: реальний server на server.port // MOCK: MockMvc без реального сервера (default) // NONE: без web layer
@SpringBootTest запускає весь контекст — повільно. Для більшості тестів краще @WebMvcTest або @DataJpaTest. @SpringBootTest — для smoke tests і критичних flow.
Testcontainers

Testcontainers — реальна DB в тестах

// build.gradle // testImplementation 'org.springframework.boot:spring-boot-testcontainers' // testImplementation 'org.testcontainers:postgresql' @SpringBootTest @Testcontainers class RepositoryIntegrationTest { @Container static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:16-alpine") .withDatabaseName("testdb") .withUsername("test") .withPassword("test"); @DynamicPropertySource static void props(DynamicPropertyRegistry r) { r.add("spring.datasource.url", postgres::getJdbcUrl); r.add("spring.datasource.username", postgres::getUsername); r.add("spring.datasource.password", postgres::getPassword); } }
Spring Boot 3.1+: @ServiceConnection замінює @DynamicPropertySource — автоматично конфігурує DataSource:
@Container @ServiceConnection
static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:16-alpine");

@DataJpaTest — тест репозиторію

Завантажує тільки JPA layer: entities, repositories. Використовує in-memory DB (H2) або Testcontainers. Кожен тест в транзакції — автоматичний rollback.

@DataJpaTest @AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.NONE) @Testcontainers class RouteRepositoryTest { @Container @ServiceConnection static PostgreSQLContainer<?> pg = new PostgreSQLContainer<>("postgres:16-alpine"); @Autowired RouteRepository repo; @Autowired TestEntityManager em; @Test void findByUserId_returnsOnlyUserRoutes() { em.persist(new Route(1L, "Kyiv tour")); em.persist(new Route(2L, "Lviv tour")); em.flush(); var result = repo.findByUserId(1L); Assertions.assertEquals(1, result.size()); Assertions.assertEquals("Kyiv tour", result.get(0).getName()); } }
Replace.NONE — використовувати реальну DB (Testcontainers) замість H2. Рекомендую завжди — H2 має інший SQL діалект і не перевірить PostgreSQL-специфічні запити.

Test slices і WireMock

Всі test slices

@WebMvcTest // Controllers, MockMvc @DataJpaTest // JPA entities, repositories @DataRedisTest // Redis layer @DataMongoTest // MongoDB layer @JsonTest // Jackson serialization @RestClientTest // RestTemplate / RestClient

WireMock — mock зовнішніх HTTP сервісів

// testImplementation 'org.wiremock.integrations:wiremock-spring-boot' @SpringBootTest @EnableWireMock class ExternalApiTest { @InjectWireMock WireMockServer wireMock; @Test void fetchPoi_returnsData() { wireMock.stubFor( WireMock.get("/pois/1") .willReturn(WireMock.okJson(""" {"id": 1, "name": "Museum"} """)) ); var poi = poiClient.fetchById(1L); Assertions.assertEquals("Museum", poi.getName()); } }
На співбесіді: піраміда (unit > integration > E2E), @WebMvcTest = web slice + @MockBean, @DataJpaTest = JPA slice + rollback, Testcontainers = реальна DB, WireMock для HTTP залежностей, Replace.NONE завжди для PostgreSQL.
