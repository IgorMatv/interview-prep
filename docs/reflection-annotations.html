reflection

Reflection API

Reflection дозволяє інспектувати і маніпулювати класами, методами, полями в runtime — навіть приватними.

// Class об'єкт Class<?> clazz = MyService.class; Class<?> clazz = obj.getClass(); Class<?> clazz = Class.forName("com.example.MyService"); // Методи Method[] methods = clazz.getDeclaredMethods(); // включно private Method m = clazz.getDeclaredMethod("doWork", String.class); m.setAccessible(true); // обхід private/protected Object result = m.invoke(instance, "arg"); // Поля Field f = clazz.getDeclaredField("value"); f.setAccessible(true); f.set(instance, "newValue"); Object val = f.get(instance); // Конструктори Constructor<?> ctor = clazz.getDeclaredConstructor(String.class); ctor.setAccessible(true); Object obj = ctor.newInstance("arg");
Spring

Як Spring використовує Reflection

  • Dependency Injection: сканує класи → знаходить @Autowired поля → field.set(bean, dependency)
  • @Transactional: під час старту знаходить анотовані методи → створює CGLIB proxy → interceptor перед/після методу
  • @RequestMapping: сканує @Controller → знаходить методи з @GetMapping → реєструє HandlerMapping
  • Jackson: сканує поля/getter'и класу → серіалізує/десеріалізує
// Спрощено: Spring DI через reflection for (Field field : bean.getClass().getDeclaredFields()) { if (field.isAnnotationPresent(Autowired.class)) { field.setAccessible(true); Object dependency = context.getBean(field.getType()); field.set(bean, dependency); } } // Спрощено: сканування @RequestMapping for (Method method : controller.getClass().getDeclaredMethods()) { GetMapping mapping = method.getAnnotation(GetMapping.class); if (mapping != null) { registerHandler(mapping.value(), method); } }
Spring робить більшість reflection-роботи при startup, кешує результати. В runtime — здебільшого CGLIB proxy (без reflection overhead). Тому Spring в продакшн швидкий попри використання reflection.
annotations

Meta-annotations

Анотації над анотаціями. Визначають де і як довго анотація існує.

// @Target: де дозволено використовувати @Target({ElementType.METHOD, ElementType.TYPE}) // TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, // LOCAL_VARIABLE, ANNOTATION_TYPE, PACKAGE, // TYPE_PARAMETER, TYPE_USE, MODULE, RECORD_COMPONENT // @Retention: скільки живе анотація @Retention(RetentionPolicy.RUNTIME) // SOURCE: тільки в вихідному коді (для apt) // CLASS: у .class файлі, але не в runtime (default) // RUNTIME: доступна через reflection в runtime // @Documented: включати в Javadoc @Documented // @Inherited: підкласи успадковують анотацію класу @Inherited // @Repeatable: можна застосовувати кілька разів @Repeatable(Roles.class) public @interface Role { String value(); }
Якщо @Retention не вказано — default CLASS. Для Spring та інших фреймворків що читають анотації в runtime — обов'язково RUNTIME.
практика

Custom annotation + AOP

Типовий патерн: анотація маркує метод, AOP aspect перехоплює виклик.

// 1. Визначення анотації @Target(ElementType.METHOD) @Retention(RetentionPolicy.RUNTIME) public @interface Timed { String value() default ""; // елементи анотації } // 2. Використання @Timed("poi-search") public List<Poi> findNearby(Location loc) { ... } // 3. AOP Aspect @Aspect @Component public class TimingAspect { @Around("@annotation(timed)") public Object time(ProceedingJoinPoint pjp, Timed timed) throws Throwable { long start = System.nanoTime(); try { return pjp.proceed(); } finally { long elapsed = System.nanoTime() - start; metrics.record(timed.value(), elapsed); } } }
APT

Annotation Processing (compile-time)

Annotation Processor — плагін компілятора що читає анотації і генерує код або перевіряє правила. Виконується під час компіляції, не в runtime.

// Приклади популярних APT: // Lombok: @Data, @Builder → генерує getters, equals, toString // MapStruct: @Mapper → генерує mapper реалізацію // Dagger/Hilt: @Inject → генерує DI код // AutoValue: @AutoValue → immutable value classes // Свій Annotation Processor @SupportedAnnotationTypes("com.example.MyAnnotation") @SupportedSourceVersion(SourceVersion.RELEASE_21) public class MyProcessor extends AbstractProcessor { @Override public boolean process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv) { for (Element el : roundEnv.getElementsAnnotatedWith(MyAnnotation.class)) { // Перевірка або генерація коду } return true; } }
APT генерує код під час компіляції — на відміну від reflection що працює в runtime. APT: нульовий runtime overhead. Retrofit + annotation processors + GraalVM Native Image — популярна комбінація.
нюанс

Reflection overhead і кешування

Reflection значно повільніший за прямий виклик, але кешування Method об'єктів мінімізує overhead.

// ПОГАНО: getDeclaredMethod() на кожен виклик void callMethod(Object obj) throws Exception { obj.getClass() .getDeclaredMethod("doWork") // lookup кожен раз .invoke(obj); } // ДОБРЕ: кешувати Method об'єкт private static final Method DO_WORK; static { try { DO_WORK = MyClass.class.getDeclaredMethod("doWork"); DO_WORK.setAccessible(true); // один раз } catch (NoSuchMethodException e) { throw new ExceptionInInitializerError(e); } } // Java 9+ альтернатива: MethodHandle (швидше ніж reflection) MethodHandle mh = MethodHandles.lookup() .findVirtual(MyClass.class, "doWork", MethodType.methodType(void.class)); mh.invoke(instance);
Java 9+

Module system і reflection

Java Module System (Java 9+) за замовчуванням забороняє reflection до non-exported пакетів. Потрібний явний дозвіл.

// module-info.java: відкрити пакет для reflection module com.example.app { exports com.example.api; // opens дозволяє reflection (відкриває для runtime) opens com.example.internal; // opens тільки для конкретного модуля opens com.example.model to com.fasterxml.jackson.databind; } // Без opens → InaccessibleObjectException при setAccessible() // Командний рядок (workaround для legacy libs): --add-opens java.base/java.lang=ALL-UNNAMED --add-opens java.base/java.util=ALL-UNNAMED
Spring Boot з Java 9+ може потребувати --add-opens для деяких бібліотек. У Spring Boot 3+ більшість проблем вирішено — але legacy deps можуть потребувати ручного налаштування.
На співбесіді: @Retention(RUNTIME) — обов'язково щоб читати в runtime, Spring DI через reflection + кешування при startup, MethodHandle vs reflection (швидше), модульна система і opens.
