Queue — три пари методів

Кожна операція доступна у двох варіантах: кидає exception або повертає спеціальне значення. Для BlockingQueue додаються ще два — блокуючий і з таймаутом.

Блокує (BQ only)put(e)take()—
Таймаут (BQ only)offer(e,t,u)poll(t,u)—
Правило: у звичайному коді завжди offer/poll/peek — повертають null/false замість exception. add/remove/element — коли порожня черга = помилка програмування.

Deque — двостороння черга

// Deque extends Queue — методи з обох боків deque.offerFirst(e) / deque.offerLast(e) deque.pollFirst() / deque.pollLast() deque.peekFirst() / deque.peekLast() // Stack через Deque (замість класу Stack — він legacy) deque.push(e) // = addFirst() deque.pop() // = removeFirst() deque.peek() // = peekFirst()

ArrayDeque і PriorityQueue

ArrayDeque

Кільцевий масив. Найшвидша реалізація черги і стеку для single-thread. O(1) амортизовано для всіх операцій. Не thread-safe, null не підтримує.

// Як FIFO черга Deque<String> q = new ArrayDeque<>(); q.offer("a"); q.offer("b"); q.poll(); // → "a" // Як LIFO стек Deque<String> s = new ArrayDeque<>(); s.push("bottom"); s.push("top"); s.pop(); // → "top"

PriorityQueue — min-heap

poll() завжди повертає найменший елемент. O(log n) для offer/poll, O(1) для peek. Не thread-safe.

// Natural order (min-heap) PriorityQueue<Integer> pq = new PriorityQueue<>(); pq.offer(5); pq.offer(1); pq.offer(3); pq.poll(); // → 1 (мінімум) // Max-heap new PriorityQueue<>(Comparator.reverseOrder()); // Custom order new PriorityQueue<>(Comparator.comparingInt(Task::priority)); // ВАЖЛИВО: ітерація — НЕ в порядку пріоритету! // Тільки poll() гарантує порядок
PriorityQueue не thread-safe. Для concurrent — PriorityBlockingQueue.

BlockingQueue — блокуючі операції

put() блокує якщо черга повна. take() блокує якщо черга порожня. Це і є механізм координації між producer і consumer — без explicit lock.

// put() — чекає місця queue.put(element); // take() — чекає елемента Element e = queue.take(); // offer/poll з таймаутом — для graceful shutdown boolean ok = queue.offer(element, 100, TimeUnit.MILLISECONDS); Element e = queue.poll(100, TimeUnit.MILLISECONDS); // null → timeout // drainTo — батч-вибірка (атомарна) List<Task> batch = new ArrayList<>(); queue.drainTo(batch); // всі елементи queue.drainTo(batch, 100); // максимум 100 // remainingCapacity() — скільки місця залишилось int free = queue.remainingCapacity();

Producer-Consumer через BlockingQueue

class Pipeline<T> { private final BlockingQueue<T> queue; private volatile boolean running = true; Pipeline(int capacity) { // ABQ — bounded, вбудований backpressure queue = new ArrayBlockingQueue<>(capacity); } // Producer: блокує якщо черга повна void produce(T item) throws InterruptedException { queue.put(item); } // Consumer: poll з таймаутом — дозволяє перевірити running void startConsuming(Consumer<T> handler) { while (running || !queue.isEmpty()) { try { T item = queue.poll(100, TimeUnit.MILLISECONDS); if (item != null) handler.accept(item); } catch (InterruptedException e) { Thread.currentThread().interrupt(); break; } } } void stop() { running = false; } }
poll(timeout) замість take() — ключовий нюанс. take() блокує вічно і не дозволяє перевірити прапор зупинки.

Poison pill — graceful shutdown

static final Task POISON = new Task(); // Producer сигналізує про кінець for (Task t : tasks) queue.put(t); queue.put(POISON); // Consumer зупиняється на маркері while (true) { Task t = queue.take(); if (t == POISON) break; process(t); } // При N consumers — N poison pills for (int i = 0; i < numConsumers; i++) queue.put(POISON);

CopyOnWriteArrayList

При кожному write (add, remove, set) — повна копія масиву. Читання і ітерація — snapshot без lock. Ніколи не кидає ConcurrentModificationException.

CopyOnWriteArrayList<Listener> listeners = new CopyOnWriteArrayList<>(); // Thread-safe ітерація — snapshot при початку for-each for (Listener l : listeners) { l.onEvent(event); // listeners.remove(l) всередині — ОК, не CME // але зміна не видима в поточній ітерації } // Типовий use case: event listeners, subscriber lists // де читань багато, додавань/видалень мало
