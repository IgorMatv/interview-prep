навіщо

Навіщо Enum — краще ніж constants

// Антипатерн: int/String constants public static final int STATUS_ACTIVE = 1; public static final int STATUS_DELETED = 2; // Проблеми: можна передати будь-який int, немає type safety // Enum: type-safe, namespace, методи public enum OrderStatus { CREATED, PAID, SHIPPED, DELIVERED, CANCELLED; } // Enum під капотом = клас що extends Enum<T> // Кожен елемент — public static final instance // Конструктор private → не можна створити додаткові // Thread-safe, serializable // Базові методи OrderStatus.PAID.name() // → "PAID" (String) OrderStatus.PAID.ordinal() // → 1 (позиція, 0-based) OrderStatus.valueOf("PAID") // → OrderStatus.PAID OrderStatus.values() // → OrderStatus[]
ordinal() — не використовуй для persistence або логіки. При додаванні нового елементу посередині — ordinal всіх наступних зміниться. Зберігай name() або власне поле.
поля і методи

Поля і методи в Enum

public enum HttpStatus { OK(200, "OK"), NOT_FOUND(404, "Not Found"), INTERNAL_ERROR(500, "Internal Server Error"); private final int code; private final String description; // Конструктор завжди private (implicit або explicit) HttpStatus(int code, String description) { this.code = code; this.description = description; } public int getCode() { return code; } public String getDescription() { return description; } // Статичний lookup public static HttpStatus fromCode(int code) { return Arrays.stream(values()) .filter(s -> s.code == code) .findFirst() .orElseThrow(() -> new IllegalArgumentException("Unknown: " + code)); } } HttpStatus.NOT_FOUND.getCode(); // → 404 HttpStatus.fromCode(200); // → HttpStatus.OK
abstract методи

Abstract методи — поведінка per constant

public enum Operation { ADD { @Override public double apply(double a, double b) { return a + b; } }, SUBTRACT { @Override public double apply(double a, double b) { return a - b; } }, MULTIPLY { @Override public double apply(double a, double b) { return a * b; } }; public abstract double apply(double a, double b); } // Замість switch/if по типу операції: double result = Operation.ADD.apply(5, 3); // → 8.0 // Нова операція → новий елемент enum, не зміна існуючого коду (OCP)
Abstract methods в Enum — реалізація Strategy патерну без окремих класів. Компілятор гарантує що кожен елемент реалізує метод.
EnumSet

EnumSet — набір enum значень

Реалізований як bit vector. Надзвичайно швидкий (O(1) для add/contains/remove) і компактний. Завжди використовуй замість Set<MyEnum>.

enum Permission { READ, WRITE, EXECUTE, DELETE } // Створення EnumSet<Permission> readOnly = EnumSet.of(Permission.READ); EnumSet<Permission> all = EnumSet.allOf(Permission.class); EnumSet<Permission> none = EnumSet.noneOf(Permission.class); EnumSet<Permission> noDelete = EnumSet.complementOf( EnumSet.of(Permission.DELETE)); // Операції readOnly.contains(Permission.WRITE); // O(1) readOnly.add(Permission.WRITE); readOnly.remove(Permission.READ); // Типовий use case: перелік дозволів, feature flags if (userPerms.containsAll(EnumSet.of(READ, WRITE))) { // має обидва дозволи }
EnumMap

EnumMap — Map з enum ключами

Реалізований як масив де index = ordinal. Швидший і компактніший ніж HashMap для enum ключів.

EnumMap<OrderStatus, String> labels = new EnumMap<>(OrderStatus.class); labels.put(OrderStatus.CREATED, "Нове замовлення"); labels.put(OrderStatus.PAID, "Оплачено"); labels.put(OrderStatus.SHIPPED, "Відправлено"); labels.put(OrderStatus.DELIVERED, "Доставлено"); // Порядок ітерації = порядок оголошення enum (гарантовано!) labels.forEach((status, label) -> System.out.println(status.ordinal() + ": " + label)); // Типовий use case: обробники per статус EnumMap<OrderStatus, Consumer<Order>> handlers = new EnumMap<>(OrderStatus.class); handlers.put(OrderStatus.PAID, order -> sendConfirmationEmail(order)); handlers.put(OrderStatus.SHIPPED, order -> notifyShipping(order)); // замість switch statement handlers.getOrDefault(order.getStatus(), o -> {}).accept(order);
Singleton

Singleton через Enum — найкращий спосіб

Enum singleton захищений від reflection, серіалізації і thread-safety "безкоштовно". Joshua Bloch рекомендує в Effective Java.

public enum DatabaseConnection { INSTANCE; private final Connection conn; DatabaseConnection() { conn = createConnection(); } public Connection getConnection() { return conn; } } // Використання DatabaseConnection.INSTANCE.getConnection(); // Переваги над звичайним Singleton: // ✓ JVM гарантує один екземпляр // ✓ Thread-safe ініціалізація // ✓ Захист від reflection (Enum.newInstance кидає exception) // ✓ Серіалізація = name(), не копіює об'єкт
state machine

State machine і нюанси Enum

public enum OrderStatus { CREATED { @Override public boolean canTransitionTo(OrderStatus next) { return next == PAID || next == CANCELLED; } }, PAID { @Override public boolean canTransitionTo(OrderStatus next) { return next == SHIPPED || next == CANCELLED; } }, SHIPPED { @Override public boolean canTransitionTo(OrderStatus next) { return next == DELIVERED; } }, DELIVERED, CANCELLED; public boolean canTransitionTo(OrderStatus next) { return false; } } // Перевірка переходу if (!current.canTransitionTo(next)) throw new IllegalStateException("Invalid transition");
На співбесіді: Enum = type-safe constants + methods + fields, ordinal() не використовувати для persistence, EnumSet = bit vector O(1), EnumMap = array by ordinal, Enum singleton = найкращий Singleton патерн (Effective Java).
